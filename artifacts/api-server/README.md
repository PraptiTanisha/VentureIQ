## Dataset

| Property | Value |
|---|---|
| Records | 2 000 synthetic businesses |
| Features | 6 (investment, team, experience, marketing, competition, market) |
| Target | Binary: success (1) / failure (0) |
| Balance | ~73% success / 27% failure |
| Split | 80% train / 20% test |

**Feature distributions:**
- `investment_amount` ‚Äî log-normal, $5 000‚Äì$2 000 000
- `team_size` ‚Äî Poisson (Œª=6), capped at 50
- `founder_experience` ‚Äî Gamma distribution, 0‚Äì25 years
- `marketing_budget` ‚Äî Beta fraction of investment (skewed low)
- `competition_level` ‚Äî categorical: Low (30%) / Medium (40%) / High (30%)
- `market_size` ‚Äî categorical: Small (25%) / Medium (45%) / Large (30%)

---

## Model

**Algorithm:** `RandomForestClassifier` (scikit-learn)

| Hyperparameter | Value |
|---|---|
| `n_estimators` | 200 |
| `max_depth` | 8 |
| `min_samples_split` | 10 |
| `min_samples_leaf` | 5 |
| `class_weight` | balanced |

**Test accuracy:** ~72%

**Feature importances (approx.):**
| Feature | Importance |
|---|---|
| marketing_budget | 0.301 |
| founder_experience | 0.230 |
| investment_amount | 0.168 |
| competition_level | 0.161 |
| market_size | 0.082 |
| team_size | 0.058 |

## Frontend Integration

The React + TypeScript frontend in `artifacts/venture-iq` calls `/api/predict` via auto-generated React Query hooks (from `@workspace/api-client-react`). The Python backend returns the exact same JSON schema, so **zero frontend changes** were needed.
